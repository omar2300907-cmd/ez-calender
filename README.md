# ez-calender
smart calendar 
📖 About the Project EZ Calendar System is a lightweight, responsive web-based application designed to allow users to manage their personal schedules efficiently. It provides a seamless "Single Page Application" feel, allowing users to switch views and manage their schedules dynamically without reloading.The system uses a role-based security layer ensuring that standard users can access their own data, while administrators have elevated privileges to maintain the user base.✨ Features🔐 Authentication: Secure user registration, login, session management, and password hashing.📅 Calendar Management: * View events in Month, Week, and Day formats.CRUD functionality: Create, Read, Update, and Delete events.Seamless Drag-and-Drop functionality for quick rescheduling using AJAX.Conflict detection logic for scheduling.Handle recurring events via series logic.⏰ Reminders: Set automated email or system notifications for upcoming events.🛡️ Admin Management: Dedicated dashboard for administrators to view, edit, or delete user accounts.🛠️ Tech Stack & ArchitectureThis project is built using a standard PHP/MySQL stack and follows a modular MVC (Model-View-Controller) architecture to ensure code maintainability and scalability.Frontend (Client): Responsive HTML, CSS, and JavaScript interface utilizing a grid-based layout and AJAX for asynchronous server requests.Backend (Server): PHP processing business logic and routing. Implements design patterns like Singleton (Database Connection) and Factory Method (User/Admin creation).Database: Relational MySQL Database handling primary and foreign key constraints for referential integrity.🚀 Getting Started (Installation)Clone the repository:Bashgit clone https://github.com/omar2300907-cmd/ez-calender.git
Database Setup:Import the provided SQL file (containing the ER schema for Users, Events, and Reminders) into your MySQL database.Configure Environment:Open the database configuration file (e.g., inside your Models/Database singleton).Update the credentials (DB_HOST, DB_USER, DB_PASS, DB_NAME) to match your local MySQL server.Run the Application:Serve the project directory using a local server like XAMPP, WAMP, or MAMP.Navigate to the project's entry point (e.g., public/index.php) in your browser.
